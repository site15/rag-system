**Подход к работе и методологии:**

В работе придерживаюсь принципов глубокой декомпозиции и построения модульных систем. Архитектуру проектирую так, чтобы модули можно было в любой момент вынести в отдельные микросервисы и масштабировать независимо.

Использую monorepo-подход на базе Nx (Nrwl).
Для работы с БД применяю Prisma ORM.

Основная СУБД — PostgreSQL. Имею большой опыт работы с Oracle DB, также работал с MySQL и MS SQL.

Для миграций БД использую Flyway. Разработал и опубликовал собственную TypeScript-реализацию в виде npm-пакета: https://www.npmjs.com/package/pg-flyway

Для взаимодействия с фронтендом и мобильными клиентами применяю генерацию SDK на основе OpenAPI или GraphQL, чтобы клиентская сторона не реализовывала транспортную логику вручную.

Микросервисы разрабатываю с использованием REST и GraphQL. Для передачи большого бинарного трафика применяю gRPC-стримы, для синхронных запросов — gRPC unary.

Локальную инфраструктуру поднимаю через docker-compose, продакшен-окружения — в Kubernetes.
CI/CD настраиваю с помощью GitLab CI/CD или GitHub Actions в зависимости от проекта.

Для всей бизнес-логики пишу E2E-тесты через TypeScript SDK.
Для сложных сценариев — unit-тесты функций и файлов без привязки к фреймворку.
Для проверки взаимодействия модулей использую интеграционные тесты.

Всегда закладываю поддержку мультиязычности и нескольких временных зон как на бэкенде, так и на фронтенде.

Для SEO-ориентированных проектов использую Angular SSR.
